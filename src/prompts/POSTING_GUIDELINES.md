# 小红书发帖指南

**重要说明**：
请严格按照以下结构和规则生成小红书笔记内容。
所有生成内容必须基于真实信息，避免夸大或虚构。



## 写作前准备


### 了解基本要求


**标题字数**
- **12-20个中文字符**（约60-100字符）
- 英文单词按1个字符计算
- **避免过短（<10字）或过长（>25字）**


**正文字数**
- **总字数**：100-300字（**不超过300字**）
- **每段长度**：30-50字（3-6行）
- **小标题频率**：每10-40字一个


**分段要求**
**流畅自然的分段原则**：
- **逻辑清晰**：每段只讲一个核心观点，避免内容混杂，每段之间空一行。
- **节奏适中**：长短段落交替，避免所有段落长度一致
- **视觉友好**：每段3-5行，避免大段文字造成阅读疲劳




### 1. 查看范文（必须）

**重要**：在开始写作前，必须先查看范文参考。通过资源查看范文
- 查看 MCP 资源列表中的 `xhs://examples/` 资源，选择相关范文进行参考
- 直接读取资源 URI：`xhs://examples/{filename}`


**范文使用原则**：
- 参考范文的结构、语气和表达方式
- 学习范文的段落划分和过渡技巧
- 借鉴范文的互动方式和结尾设计
- **不要直接复制**，要根据当前主题进行原创写作




### 2. 获取人设信息（必须）

**重要**：在生成内容前，必须先调用 `xhs_get_my_profile` 工具获取当前登录用户的资料信息，包括：
- 账户名称
- 粉丝数量
- 关注数量
- 获赞与收藏数
- 小红书账号ID
- 个人描述
- 账户状态

**人设使用原则**：
1. **基于真实资料**：根据获取的用户资料信息，确定内容的人设和身份定位
2. **语气与风格**：
   - **专业但亲切**：像朋友分享经验，而不是销售推销
   - **真实可信**：使用第一人称，分享真实经历和个人感悟
   - **真诚有人情味**：避免模板化语言，加入个人思考和情感
   - **价值导向**：每篇文章必须提供可操作的方法或工具，真正帮助用户解决问题
   - **互动性强**：结尾设置开放性问题，引导真实讨论
3. **内容适配**：根据用户的专业领域、粉丝规模、账户定位等信息，调整内容的专业深度和表达方式



### 图文混排说明


**重要**：小红书不支持图文混排功能。


**注意事项**：
- **不要**在正文中插入图片标记或占位符（如 `[图片1]`、`[配图]` 等）
- **不要**在正文中描述图片内容或位置
- **不要**使用"上图"、"下图"、"配图"等指向图片的词汇
- 图片由系统单独上传，与正文内容分离
- 正文只需专注于文字内容的流畅性和完整性

**正确的做法**：
- 正文内容完整独立，不依赖图片说明
- 如果内容需要配图，在生成内容后单独处理图片上传
- 正文中可以用文字描述场景，但不要提及图片
